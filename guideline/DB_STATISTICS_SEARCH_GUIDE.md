# DB 통계 및 검색 기술 가이드라인 (DB_STATISTICS_AND_SEARCH_GUIDE.md)

이 문서는 **DB 통계 및 검색 탭**의 SQLite3 데이터베이스 아키텍처, 테이블 스키마, 통합 검색 모듈 구조 및 쿼리 활용법을 설명합니다.

---

## 1. 🗄️ SQLite 데이터베이스 스키마 명세

대시보드와 검색 및 API 동기화 레이어가 통합 공유하는 SQLite3 데이터베이스(`youtube_data.db`)의 테이블 구조는 다음과 같습니다.

### 1.1. shorts_rank 및 videos_rank (쇼츠/일반 영상 랭킹 이력)
각각 쇼츠 카테고리 랭킹과 일반 비디오 랭킹의 데이터 이력을 저장합니다.
- `UNIQUE(video_id, category, country, period, crawled_at)` 제약조건을 걸어 동일 영상이 다양한 국가/카테고리/주기 차트에 진입했을 때의 데이터 무결성을 보장합니다.

### 1.2. channels_rank (채널 인기 및 급상승 랭킹)
인기 순위 및 급상승 랭킹 데이터를 수집합니다.
- **주요 속성**:
  - `channel_url`: Playboard 채널 상세 주소 (Zero-Cost ID Extraction의 소스가 됨)
  - `ranking_type`: 랭킹 유형 구분 (`popular`: 인기순위, `growth`: 구독자급상승)
  - `subscriber_count`: 구독자 수 문자열 (예: "1.2M", "500K")
  - `total_views`: 채널 전체 누적 조회수

### 1.3. api_channels (YouTube API 연동 채널 데이터)
- **역할**: Zero-Cost ID Extraction으로 추출한 Channel ID를 통해 YouTube API v3에서 수집한 상세 속성을 저장합니다.
- **주요 속성**:
  - `uploads_playlist_id`: 채널의 모든 업로드 비디오 목록 재생목록 ID (영상 동기화 시 필수 참조)

### 1.4. api_videos (YouTube API 연동 영상 데이터)
- **역할**: YouTube API를 통해 수집된 채널 소속 동영상들의 상세 수치 및 재생 시간 정보를 분류 보관합니다.
- **주요 속성**:
  - `duration_sec`: 재생 시간(초) 환산 정수 (검색/필터링용)
  - `video_type`: 재생 시간이 60초 이하이면 `shorts`, 그렇지 않으면 `video`로 자동 판별 및 저장

## 2. 📊 크롤링 데이터 대시보드 및 상세 검색

대시보드 메인페이지의 **크롤링 데이터** 탭은 수집된 원본 크롤링 데이터를 일자별로 브라우징하고 검색할 수 있는 기능을 제공합니다.

### 2.1. 일자별 대시보드 뷰 (Thumbnail Dashboard)
* **날짜 활성화**: 데이터베이스 (`shorts_rank`, `videos_rank`, `channels_rank`) 내에 실제 수집된 이력이 있는 날짜만 옵션으로 활성화하여 제공합니다.
* **버튼 나열형 다차원 필터 (st.pills)**:
  - 드롭다운을 열 필요 없이 클릭 한 번으로 모든 조건을 즉시 토글할 수 있는 가로 버튼 나열형 컴포넌트를 사용합니다.
  - **고대비 버튼 및 크기 확대**: 알약 버튼(`st.pills`)의 크기와 패딩을 15% 이상 늘리고, 활성화 시 **푸른색 단색 배경**(`#0056b3`)과 글로우 효과가 가미되어 비활성화 상태의 버튼과 매우 뚜렷한 시각적 대비를 제공합니다. `st.radio` 라벨도 확대하여 시인성을 강화했습니다.
  - **수집 기준 (Criteria)**: 조회수 순위, 좋아요 순위, 댓글 순위 버튼을 가로로 제공하며, 클릭 시 해당 조건의 정렬 및 랭킹 메타 정보가 대시보드에 실시간 반영됩니다.
  - **국가 (Country) / 기간 (Period)**: 탭 1에서 선택한 수집 설정을 기본 바인딩하되, 탭 2 내에서도 직접 버튼을 클릭하여 임의 조건의 데이터를 빠르게 필터링할 수 있습니다.
  - **카테고리 선택**: 데이터 구분 하단에 데이터셋에 존재하는 카테고리들을 가로 알약 버튼 형태로 노출합니다. 맨 앞에 `"All"` 알약이 제공되며 그 뒤에 실제 수집된 카테고리들과 플레이보드 종합 차트인 `"전체"` 카테고리가 각각 분리되어 렌더링됩니다.
  - **All 및 전체 카테고리 구분**: `"All"`을 클릭하면 모든 카테고리 데이터를 모아서 보여주고, `"전체"`를 클릭하면 플레이보드 종합 차트 전체 카테고리 데이터만 단독 렌더링하도록 거동을 차별화했습니다.
* **정렬 순서 (st.selectbox)**: 날짜별 대시보드 뷰 우측 상단에 정렬 기준 셀렉트박스를 제공합니다.
  - `높은 순위순 / 낮은 순위순`: 랭킹 순위(1위~100위)를 기준으로 오름차순/내림차순 정렬하여 보여줍니다. `"All"` 카테고리 선택 시에는 각 카테고리별로 `expander`를 분리하여 표시합니다.
  - `지표 높은순 / 지표 낮은순`: 조회수, 좋아요, 댓글 등 선택한 수집 기준의 수치를 기준으로 정렬합니다. `"All"` 카테고리 선택 시에는 카테고리 경계 없이 모든 수집 영상 데이터를 통합 병합하여 지표 수치순으로 일렬 정렬 노출합니다.
* **구분별 브라우징**: `쇼츠 (Shorts)`, `일반 영상 (Video)`, `채널 (Channel)`을 라디오 버튼으로 분류하여 제공합니다.
* **레이아웃 스타일 선택**:
  - `캐러셀형 (카드 그리드)`: 4열 반응형 그리드 형태의 카드 뷰. 리스트형과 스타일의 조화를 위해 카드 내 채널명을 **형광 파란색**(`#00d2ff`)으로 일치시키고 지표 정보의 글자색을 **흰색**(`#ffffff`)으로 고대비 적용했습니다. 순위(1.05em), 제목(1.25em) 등 폰트 크기를 크게 키우고, 제목/채널명 마우스 오버 시 밑줄 효과와 손가락 커서(`pointer`)를 표시하여 클릭 시 즉시 클립보드에 텍스트가 복사되는 원클릭 인터랙션을 전면 적용했습니다.
  - `리스트형 (썸네일 포함)`: 카드의 배경색은 차분하고 깊이 있는 어두운 회색(`#1e1e24`)으로 유지하고, 왼쪽의 **순위 영역 뱃지**만 고대비를 이루는 **단색 푸른색**(`#0056b3`) 배경을 적용하여 시인성을 극대화합니다. 썸네일 크기를 **240px x 135px**로 확대하고, 순위(1.4em), 제목(1.4em), 채널명(1.1em, 채널명 접두어 고정), 조회수(1.1em, 흰색) 등 가독성을 높였습니다. 또한 제목 및 채널명을 클릭하면 클립보드에 복사(알림 팝업 노출)되는 인터랙션을 지원합니다.



### 2.2. 크롤링 상세 검색 필터 매개변수
* **검색 키워드**: 제목, 채널명에 대한 와일드카드 매칭
* **컨텐츠 타입**: 전체, shorts, video, channel
* **정렬 기준**: 최근 등록일순, 조회수 높은순, 높은 순위순, 낮은 순위순 (순위 기반 1위부터/100위부터의 정렬 쿼리 지원)
* **고급 필터**: 최소/최대 조회수 임계값 설정

### 2.3. 크롤링 데이터 시각화 분석
* **Shorts 수집 카테고리 분포**: 카테고리별 랭킹 수집 비중을 원형 파이 차트(Pie Chart)로 표현
* **인기 채널 TOP 10**: 구독자수 기준 상위 10개 채널을 가로 막대 차트(Bar Chart)로 시각화

## 3. 🔍 API 연동데이터 상세 검색 및 시각화

**API 연동데이터** 탭은 YouTube API v3를 활용해 수집된 채널 및 동영상의 정보를 심층 탐색합니다.

### 3.1. API 데이터 상세 검색 필터 매개변수
* **검색 키워드**: 제목, 채널명, 태그에 대한 와일드카드 검색
* **비디오 타입**: 전체, shorts, video
* **정렬 기준**: 최근 등록일순, 조회수 높은순, 좋아요 비율 높은순 (조회수 대비 좋아요 수 백분율 자동 계산 및 정렬)
* **고급 필터**: 최소/최대 조회수 필터, 게시일 범위 지정 필터

### 3.2. API 데이터 시각화 분석
* **연동 비디오 타입 비율**: 수집된 영상들의 쇼츠와 일반 영상 비율을 나타내는 도넛 파이 차트
* **API 연동 비디오 카테고리 분포**: YouTube API에서 판정된 카테고리별 동영상 수 분포 막대 차트

---

## 4. ⚙️ 검색 설정값 자동 보존 (Query Parameters Auto-Persistence)

각 탭 내 상세 검색 필터의 사용성 극대화를 위해 사용자가 필터링한 검색 키워드, 선택한 정렬 기준, 최소/최대 조회수, 게시일 날짜 등의 값들은 변경되는 즉시 `output/settings.json` 파일에 스마트 직렬화 저장됩니다.
이를 통해 대시보드를 새로고침하거나 브라우저를 재부팅해도 이전에 사용하던 검색 필터 상태가 그대로 복구되어 편의성을 제공합니다.
