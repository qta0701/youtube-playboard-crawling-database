# 외부 프로그램 대시보드 탭 통합 가이드라인 (EXTERNAL_PROGRAMS_INTEGRATION_GUIDE.md)

이 문서는 `외부프로그램\` 폴더에 존재하던 독립형 프로젝트(대본 추출기 및 유튜브 검색기)가 Streamlit 대시보드(`app.py`)에 통합된 아키텍처와 구동 방식에 대해 설명합니다.

---

## 1. 아키텍처 설계 (Architecture Design)

롱폼과 숏폼의 외부 프로그램은 기존에 각 폴더에서 개별적으로 구동되었으며, 중복된 모듈 이름(`sheet_config`, `sheet_utils` 등)을 가지고 있었습니다. 이를 그대로 `sys.path.append`로 로드하면 네임스페이스 충돌로 인해 잘못된 구글 시트 칼럼 매핑이 일어나는 문제가 있었습니다.

이를 방지하기 위해 **격리 모듈 동적 로더(Isolated Module Loader)**를 설계하여 동적으로 각 외부 프로그램을 격리 실행합니다.

### 격리 로딩 원리 (`modules/external_loader.py`)
1. **sys.path 샌드박싱**: 호출하는 외부 프로그램의 루트 폴더 경로를 `sys.path` 최상단에 임시 삽입하여 로컬 임포트의 일치성을 보장합니다.
2. **sys.modules 샌드박싱**: 충돌이 우려되는 모듈(`sheet_config`, `sheet_utils` 등)이 기존에 캐싱되어 있다면 임시 백업한 후 `sys.modules`에서 제거합니다.
3. **독립 모듈 생성**: `importlib.util.spec_from_file_location`을 통해 런타임에 새로운 모듈 객체를 샌드박스에서 로드합니다.
4. **sys.modules 복원**: 모듈 로드가 완료되면 백업해 두었던 원본 모듈과 `sys.path`를 원 상태로 완벽히 복구합니다.

## 2. 📝 대본(Transcript) 자동 추출기 통합 (Tab 5)

대본 자동 추출기는 롱폼 및 숏폼 대본 추출 프로젝트를 통합하여 단일 인터페이스로 제공합니다.

### 주요 기능 및 매개변수
- **추출기 종류**: 롱폼 대본 추출기 / 숏폼 대본 추출기 라디오 버튼으로 제어합니다.
- **스프레드시트 URL**: 자주 쓰이는 구글 스프레드시트 주소를 프리셋 드롭다운 메뉴로 선택하거나 직접 텍스트 필드에 입력할 수 있습니다.
- **실행 모드**:
  - `모드 A (전체 추출)`: 10행부터 시작하여 대본유무가 비어 있는 모든 행을 순차 처리합니다.
  - `모드 B (마지막 대본 이후)`: 대본이 기입된 마지막 행 번호 이후의 비어 있는 행들을 대상으로 처리합니다.
  - `모드 C (중간 누락 대본 추출)`: 10행부터 끝까지 훑어가며 비어 있는 대본 셀들만 선별적으로 추출을 실행합니다.
- **성능/고급 옵션**: Concurrency(동시 처리수), Delay(지연 시간), Browser Auto(셀레늄 사용 여부), Headless(창 감춤 여부), Chrome Profile 적용 여부를 제어할 수 있습니다.

### 실시간 피드백 및 제어
- 백그라운드 Worker 스레드(`run_extractor_work`)를 가동하여 Streamlit 메인 UI 스레드와 통신합니다.
- `StreamlitLogHandler`를 통해 외부 로거의 메시지를 캡처하고 실시간 현황판에 출력합니다.
- `🛑 추출 프로세스 중단` 버튼을 통해 진행 중인 루프에서 즉시 안전하게 추출기 브라우저를 닫고 보존합니다.

## 3. 🔎 유튜브 키워드 검색기 통합 (Tab 6)

유튜브 검색기는 YouTube Data API v3를 활용하여 키워드 매칭 비디오를 탐색하고, 구글 스프레드시트에 서식 붕괴 없이 누적 벌크 저장합니다.

### 주요 기능 및 매개변수
- **정렬 기준**: `relevance` (관련성), `date` (최신성), `viewCount` (조회수), `rating` (평점) 등을 지정 가능합니다.
- **영상 길이 필터**: `any` (전체), `short` (4분 이하), `medium` (4-20분), `long` (20분 이상) 필터링이 가능합니다.
- **구글 시트 벌크 저장**: 10행부터 순차적으로 저장하며, 구글 시트의 기존 수식 열(예: 계산된 지표 등)을 훼손하지 않도록 서식 자동 보호 연동이 수행됩니다.
- **OAuth 및 API Key 인증**: GCP 연동 client_secret JSON 파일이 있는 경우 OAuth 사용자 인증을 자동으로 진행하고, 없을 시 api_key 텍스트를 감지하여 유연하게 가동합니다.

## 4. 알림 및 로그 정책 (Sound & Notification Policy)

`rules.md` 정책을 완벽히 수용하여 다음과 같이 구축되었습니다.
- **작업 완료 또는 중단/실패 알림**:
  - 윈도우 OS의 기본 알림음 `%SystemRoot%\Media\Speech Off.wav` 사운드를 비동기적으로 자동 재생(`winsound.PlaySound` 활용)합니다.
  - 외부 라이브러리 의존성 없이 PowerShell을 활용하여 **OS Toast/Balloon 팝업 알림**을 생성함으로써 사용자 환경에서 성공/실패 여부를 윈도우 알림 센터로 실시간 전달합니다.
- **디버깅 로그 보존**:
  - `logs/` 디렉토리에 각 구동 타임스탬프와 매핑된 실행 파일 로그가 백업되며, 대시보드는 재실행 시 기존 로그를 무차별 초기화하지 않고 **최근 5개의 로그를 보존**하는 정밀 클린업 로직을 수행합니다.
