# 구글 스프레드시트 및 데이터베이스 통합 연동 가이드라인 (DATABASE_SHEET_INTEGRATION_GUIDE.md)

본 가이드라인은 프로젝트 내의 구글 스프레드시트 7개 탭과 로컬 SQLite3 데이터베이스 간의 양방향 벌크 동기화 연동 체계, 동적 헤더 및 중복 방지 제어 로직, 파이썬 기반 통계 자동 연산기 명세를 기술합니다.

## 2. 9행 수식 보호 및 탭별 시작 행 감지 기법

구글 시트의 상단에는 수식 및 대시보드 통계 메타데이터가 존재합니다. 시트의 서식과 수식을 훼손하지 않기 위해 탭별 시작 오프셋을 자동 감지하여 양방향 동기화를 제어합니다.

### 1) 탭별 헤더 및 데이터 오프셋 정의

* **`재생목록ID` 탭**:
  - **헤더 위치**: 1행 (1-indexed)
  - **데이터 시작 위치**: 2행부터 (0-indexed 1)
  - **클리어 및 기입 범위**: `A2:ZZ50000`
* **나머지 탭들 (영상 관련 5개 탭 및 채널 리스트 탭)**:
  - **헤더 위치**: 9행 (1-indexed)
  - **데이터 시작 위치**: 10행부터 (0-indexed 9)
  - **클리어 및 기입 범위**: `A10:ZZ50000` (9행의 본래 통계 수식 포맷은 완벽하게 보호 및 보존됨)

## 3. 동적 넘버링 매칭 엔진 (`modules/utils.py`)

구글 시트의 헤더명(예: `01. 영상 ID`, `1. 영상 ID`, `영상 ID`, `04. 검색 키워드`)은 관리자의 필요에 따라 앞에 붙은 넘버링이 유동적으로 변할 수 있습니다. 이를 안전하게 처리하기 위해 정규화 기법을 적용합니다.

### 1) 정규화 규칙 (`normalize_header_name`)
1. 선두 넘버링 패턴 제거: 정규식 `^\d+[\.\s\-]*`을 사용하여 숫자로 시작하는 일련번호와 기호를 잘라냅니다.
2. 특수문자 및 공백 제거: 모든 띄어쓰기(`\s`), 쉼표(`,`), 물음표(`?`), 괄호(`()`), 하이픈(`-`), 언더바(`_`) 등을 완전 제거합니다.
3. 알파벳 소문자화: 대소문자 구분을 없앱니다.
   - *예시*: `"31. 조회수 1,000만 이상"` $\rightarrow$ `"조회수1000만이상"` $\rightarrow$ `"views_over_10m"` 매핑 성공.

## 4. 파이썬 통계 자동 계산기 명세

유튜브 API 또는 크롤러 수집 시점에, 9행 수식이 하던 파생 연산을 파이썬 코드 (`calculate_sheet_video_metrics`)를 통해 실시간으로 계산해 DB에 적재합니다.

* **업로드 후 경과일 (`days_since_upload`)**: `(수집날짜 - 업로드날짜)`
* **일평균 조회수 (`daily_avg_views`)**: `조회수 / max(1, 경과일)` (소수 둘째자리 반올림)
* **조회수 구간 여부 (`views_over_1m` 등)**: 조회수가 각각 $1\text{M}$ / $5\text{M}$ / $10\text{M}$ 이상이면 `"ㅇ"`, 미만이면 `"x"`
* **구독자 대비 조회수 배율 (`views_multiplier`)**: `조회수 / max(1, 구독자수)`
* **좋아요 3% 이상 여부 (`likes_over_3pct`)**: `(좋아요 수 / 조회수) >= 0.03` 이면 `"ㅇ"`, 미만이면 `"x"`
* **대본 글자수 (`transcript_char_count`)**: `len(대본내용)`

## 5. 중복 데이터 방지 및 API Quota 최적화 로직

유튜브 검색 기능을 활용할 때 중복 수집 및 불필요한 Quota 소모를 방지하는 방어 체계가 가동됩니다.

### 1) 유튜브 키워드 검색 및 저장 시 중복 체크 (`app.py` - `run_search_work`)
- 대시보드의 **`🔎 유튜브 검색기`** 탭을 통해 검색 실행 시:
  1. 검색 결과 리스트에서 로컬 DB의 `sheet_videos` 테이블에 이미 적재된 비디오 ID는 **자동으로 걸러냅니다 (패스 처리)**.
  2. 필터링된 순수 신규 비디오 데이터만 구글 시트의 **`키워드 검색결과`** 탭(10행 이하)에 벌크 기입합니다.
  3. 동시에, 로컬 DB `sheet_videos` 테이블의 `'키워드 검색결과'` 영역(`tab_name='키워드 검색결과'`)에도 동적 헤더 매칭 및 통계 자동 계산을 거쳐 **중복 없이 UPSERT 적재**함으로써 시트와 DB 간의 실시간 정합성을 유지합니다.

## 6. 대시보드 테이블 뷰 컬럼 정렬 및 선택적 셀 편집 기능

대시보드 상에서 데이터를 효율적이고 안전하게 제어하기 위해 뷰어 정렬 및 잠금 장치가 구현되어 있습니다.

### 1) '채널삭제 여부' 컬럼의 강제 정렬
* '채널 리스트' 탭 데이터를 렌더링할 때, 구글 시트의 헤더 넘버링 상태와 무관하게 **'가져왔는지 여부' 컬럼의 바로 오른쪽**에 **'채널삭제 여부'** 컬럼이 위치하도록 컬럼 정렬 순서를 강제로 보정하여 가독성을 높입니다.

### 2) 'st.data_editor'를 활용한 특정 컬럼 수정 및 편집 잠금
* 대시보드 상에서 영상/채널 상태를 직접 변경할 수 있도록 셀 편집이 활성화됩니다.
* 이때 원본 메타데이터(조회수, 구독자, 제목 등)의 오입력을 방지하기 위해 컬럼명에 **`"여부"`, `"유무"`, `"가져올 채널"`** 텍스트가 포함된 컬럼만 수정을 허용하며, 그 외의 모든 컬럼은 잠금(`disabled`) 처리하여 완벽히 보호합니다.

### 3) 실시간 DB 반영 및 수집 날짜(crawl_date) 강제 갱신
* 사용자가 셀의 값을 변경하는 즉시 로컬 DB에 수정 내역이 즉각 반영(`UPDATE`)됩니다.
* 구글 시트로 보내기(`sync_db_to_sheet`) 실행 시 최신 우선 업데이트 규칙(`is_new_date_newer`에 의한 날짜 비교 조건)을 만족하여 수정 내용이 누락 없이 스프레드시트에 기입되도록, 수정이 완료되는 즉시 해당 행의 수집 날짜(`crawl_date` 또는 `last_checked_at`)를 **현재 날짜/시간**(`datetime.now()`)으로 강제 갱신합니다.

## 7. 최근 스키마 변경 사항 및 마이그레이션 이력

### 1) 채널삭제 여부 (`is_deleted_channel`) 컬럼 변경 및 반영
- **배경**: 기존 채널 리스트 DB의 불필요한 '자막 여부'(`has_transcript`) 컬럼을 비활성화하고, 채널의 삭제 상태를 실시간 기록/관리하기 위한 **'채널삭제 여부'** 컬럼으로 통합 대체하였습니다.
- **적용 테이블**:
  - `sheet_channels`: 기존 `has_transcript` 컬럼이 `is_deleted_channel` (TEXT)로 전면 대체 및 변경되었습니다.
  - `sheet_videos`: 영상 관련 5개 통합 탭 테이블에도 `is_deleted_channel` (TEXT) 컬럼이 새롭게 추가되어, 수집된 개별 영상들이 삭제된 채널에 속해 있는지 교차 진단할 수 있도록 확장되었습니다.
- **마이그레이션 안전성**: `DatabaseHandler._migrate_db()` 구동 시, 구버전 DB에 해당 신규 컬럼이 누락되어 있을 경우 SQLite `ALTER TABLE` 명령어를 자동 수행하여 데이터 유실 없는 무중단 스키마 마이그레이션을 지원합니다.

## 8. 유튜브 API 수집 탭 및 실시간 통계 최신화 로직

대시보드 상에 추가된 **`📥 채널 및 영상 수집`** 탭을 통해 유튜브 API로부터 채널 및 영상 데이터를 직접 수집하여 로컬 연동 DB에 반영할 수 있습니다.

### 1) 채널 수집 및 영상 수집 연동
- **채널 수집**: 입력한 채널명, 핸들, URL을 분석하여 채널 정보를 `sheet_channels`에 반영합니다. 이때 "영상 수집 옵션 연동"이 활성화되면, 지정한 영상 범위 설정에 따라 해당 채널의 동영상들도 연속적으로 자동 수집합니다.
- **영상 및 재생목록 수집**: 특정 단일 영상 URL, 재생목록 URL, 혹은 채널 식별값을 입력하고 수집할 수 있습니다. 수집된 영상 데이터는 `sheet_videos` 테이블에 `tab_name = '영상 리스트'` 명의로 안전하게 UPSERT 적재됩니다.
  1. *해당 영상만 수집*: 단일 영상 정보를 상세 파싱해 1개 행으로 적재합니다.
  2. *최근 영상 지정 수집*: 지정한 개수(50개 단위)만큼 최신 동영상을 수집합니다.
  3. *전체 영상 수집*: 채널의 모든 영상을 수집합니다.
  4. *지정 연도부터 수집*: 지정 연도(예: 2024년) 이후에 업로드된 영상만 수집하며, API 호출 루프 도중 해당 연도 이전의 영상이 발견되면 **수집 루프를 즉시 중단(break)**하여 Quota 소모를 최소화합니다.

### 2) 실시간 채널 통계 최신화 (`update_channel_metrics`)
영상 수집 작업이 성공적으로 완료되면, 로컬 DB 상에 축적된 동일 채널의 전체 영상 데이터를 기반으로 아래 지표들을 즉시 재계산하여 `sheet_channels` 테이블의 해당 채널 행에 실시간 반영합니다.
- **수집영상 갯수 (`collected_video_count`)**: `COUNT(*)`
- **수집한 영상 평균 조회수 (`collected_video_avg_views`)**: `AVG(views)`
- **최근 30개 영상 평균 조회수 (`avg_views_30`)**: 최신 30개 영상의 평균 조회수
- **상위 3개 제외 평균 조회수 (`avg_views_exclude_top3`)**: 가장 조회수가 높은 상위 3개 영상을 제외한 나머지 영상들의 평균 조회수
