# YouTube Crawler Pro - 사용 가이드

## 빠른 시작 (Quick Start)

### 1. 설치
```bash
# 방법 1: 배치 파일 실행 (권장)
INSTALL.bat 더블클릭

# 방법 2: 수동 설치
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 실행
```bash
# 방법 1: 배치 파일 실행 (권장)
START_SERVER.bat 더블클릭

# 방법 2: Python 직접 실행
python run.py

# 방법 3: 가상환경에서 실행
venv\Scripts\activate
python run.py
```

### 3. 브라우저 접속
- 자동으로 브라우저가 열립니다
- 수동: `http://localhost:5000` 접속

## 주요 기능

### 🎯 1. 단일 크롤링
특정 카테고리의 데이터를 수집합니다.

**사용법:**
1. **타겟 선택**: 쇼츠 / 영상 / 채널
2. **카테고리 선택**: 전체, 게임, 음악 등
3. **국가 선택**: 한국, 미국, 일본 등
4. **기간 선택**: 일간, 주간, 월간
5. **크롤링 시작** 버튼 클릭

**옵션:**
- ✅ 로그인 모드: 200개 수집 (체크 해제 시 100개)
- 📅 특정 날짜: 과거 날짜의 랭킹 조회 가능

### 📦 2. 일괄 크롤링 (NEW!)
선택한 타겟의 **모든 카테고리**를 자동으로 크롤링합니다.

**사용법:**
1. **타겟 선택**: 쇼츠 / 영상 / 채널
2. **국가 선택**: 한국, 미국, 일본 등
3. **기간 선택**: 일간, 주간, 월간
4. **📦 일괄 크롤링 (전체 카테고리)** 버튼 클릭
5. 확인 메시지에서 "확인" 클릭

**특징:**
- 13개 카테고리를 자동으로 순차 크롤링
- 각 카테고리별로 별도 CSV 파일 저장
- 실시간 진행 상황 확인 가능
- 실패한 카테고리는 건너뛰고 계속 진행

**파일명 형식:**
```
batch_shorts_게임_한국_일간_20251204_143025.csv
batch_shorts_음악_한국_일간_20251204_143310.csv
...
```

### 📝 3. 자막 추출
수집한 영상의 자막을 추출합니다.

**사용법:**
1. 크롤링 완료 후 자동으로 사용 가능
2. 또는 CSV 파일 업로드
3. **자막 추출 시작** 버튼 클릭

**출력:**
- `output/transcripts/` 폴더에 텍스트 파일 저장
- 파일명: `{video_id}_transcript.txt`

## 로그 시스템

### 📊 로그 위치
모든 로그는 `logs/` 폴더에 자동 저장됩니다.

```
logs/
├── debug_20251204.log      # 디버그 로그 (모든 상세 정보)
├── app_20251204.log         # 일반 로그 (INFO 레벨)
└── error_20251204.log       # 에러 로그 (ERROR 레벨만)
```

### 📝 로그 확인 방법

**1. 콘솔 실시간 로그**
- `START_SERVER.bat` 실행 시 콘솔에 실시간 표시
- INFO 레벨 이상만 표시

**2. 파일 로그**
```bash
# 최신 디버그 로그 확인 (Windows)
type logs\debug_*.log | more

# 에러 로그만 확인
type logs\error_*.log

# 실시간 로그 모니터링 (PowerShell)
Get-Content logs\debug_*.log -Wait -Tail 50
```

**3. 웹 UI 상태 로그**
- 화면 하단 "상태 로그" 패널
- 실시간 업데이트 (3초마다)
- 최근 20개 로그 표시

### 🔍 로그 레벨

| 레벨 | 설명 | 예시 |
|------|------|------|
| DEBUG | 상세한 진단 정보 | 함수 호출, 변수 값 |
| INFO | 일반 정보 | 크롤링 시작/완료 |
| WARNING | 경고 메시지 | 라이브러리 누락 |
| ERROR | 오류 발생 | 크롤링 실패 |

### 🐛 디버깅 팁

**문제 발생 시:**
1. `logs/error_*.log` 파일 확인
2. 에러 메시지와 스택 트레이스 확인
3. `logs/debug_*.log`에서 상세 정보 확인

**주요 로그 패턴:**
```
[2025-12-04 14:30:25] INFO     [app:crawl:95] - 크롤링 시작: shorts / 게임 / 한국 / 일간
[2025-12-04 14:30:30] DEBUG    [crawler:_init_driver:35] - Calling function: _init_driver()
[2025-12-04 14:32:15] INFO     [crawler:crawl:150] - Successfully crawled 100 items
[2025-12-04 14:32:15] ERROR    [app:crawl:139] - Exception occurred: 크롤링
```

## 트러블슈팅

### ❌ 오류: ModuleNotFoundError
```
ModuleNotFoundError: No module named 'flask'
```
**해결:**
```bash
pip install -r requirements.txt
```

### ❌ 오류: ChromeDriver not found
```
WebDriverException: 'chromedriver' executable needs to be in PATH
```
**해결:**
1. [ChromeDriver 다운로드](https://chromedriver.chromium.org/)
2. Chrome 버전 확인: `chrome://version`
3. 일치하는 버전의 ChromeDriver 다운로드
4. 시스템 PATH에 추가 또는 프로젝트 폴더에 복사

### ❌ 오류: 크롤링 0개 수집
**원인:**
- Playboard 사이트 구조 변경
- 네트워크 오류
- 봇 탐지

**해결:**
1. `logs/debug_*.log` 확인
2. 수동으로 Playboard 접속 테스트
3. `CHROME_HEADLESS = False` 설정 (config.py)
4. 브라우저 창에서 직접 확인

### ❌ 오류: 일괄 크롤링 중 멈춤
**원인:**
- 특정 카테고리에서 오류 발생
- 메모리 부족

**해결:**
1. 실패한 카테고리 확인 (로그 또는 UI)
2. 해당 카테고리만 단일 크롤링으로 재시도
3. 브라우저 창 모두 닫고 재시작

### ❌ 오류: 자막 추출 실패
```
No transcript found for video {video_id}
```
**원인:**
- 해당 영상에 자막 없음
- 자막이 비활성화됨
- API 할당량 초과

**해결:**
- 정상 (일부 영상은 자막 없음)
- 다른 영상 시도
- API 할당량 확인 (Google Cloud Console)

## FAQ

**Q: 로그인 없이 200개 수집 가능한가요?**
A: 불가능합니다. Playboard는 비로그인 시 100개로 제한합니다.

**Q: 일괄 크롤링은 얼마나 걸리나요?**
A: 13개 카테고리 × 약 2분 = 약 26분 소요 (네트워크 상태에 따라 다름)

**Q: 특정 날짜의 과거 랭킹도 수집 가능한가요?**
A: 가능합니다. "특정 날짜" 필드에 날짜를 입력하세요.

**Q: 자막이 없는 영상은 어떻게 되나요?**
A: 자동으로 건너뛰고 다음 영상으로 진행합니다. 로그에 기록됩니다.

**Q: CSV 파일 인코딩이 깨져요.**
A: 모든 파일은 UTF-8 BOM으로 저장되어 Excel에서 정상 표시됩니다.

**Q: 일괄 크롤링 중 중단하려면?**
A: 브라우저 창에서 `Ctrl+C` 또는 콘솔 창 닫기
